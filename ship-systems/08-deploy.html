Module 08: Deploy

What You Are Building

A live deployed version of your AI system, accessible at a public URL. When this module is done, anyone with the link can use your system. No "it works on my machine."

Why It Matters

If it is not deployed, it is not shipped. A system running on localhost is a prototype. A system running at a URL is a product. Deployment is the line between "I built something" and "I shipped something." It also forces you to handle real-world concerns: environment variables, dependency management, cold starts, and network behavior. These are the things that separate a project from production.

Step-by-Step Instructions

Step 1: Choose Your Platform

Pick one deployment platform. Use this table to decide based on what you built.

Platform Best For Free Tier Deploy Time
Vercel Next.js apps, static sites, serverless functions Yes ~2 min
Railway Full-stack apps, Python APIs, background workers Yes (trial) ~5 min
Fly.io Docker containers, global deployment Yes ~5 min
Render Web services, Python/Node APIs, static sites Yes ~5 min
Replit Quick deploys, any language, minimal config Yes ~1 min

If you built a CLI-only system in Module 07, you need to wrap it in an API endpoint before deploying. A CLI cannot be accessed via URL. Go back and add a minimal API server using the API starter code from Module 07, then return here.

Step 2: Set Up Your Account and CLI

Create an account on your chosen platform (if you do not already have one), then install the platform's CLI tool.

# Vercel
npm i -g vercel
vercel login

# Railway
npm i -g @railway/cli
railway login

# Fly.io
curl -L https://fly.io/install.sh | sh
fly auth login

# Render
# No CLI needed — connect your GitHub repo at render.com

# Replit
# No CLI needed — import your repo at replit.com

Step 3: Configure Your Project

Each platform needs to know how to build and start your project. Make sure these files exist in your repo root.

For Python projects:

# requirements.txt — list all dependencies
fastapi==0.104.1
uvicorn==0.24.0
openai==1.6.0
# ... your other dependencies

# Procfile (for Railway/Render)
web: uvicorn server:app --host 0.0.0.0 --port $PORT

For Node projects:

// In package.json, confirm these fields:
{
  "scripts": {
    "start": "node server.js",
    "build": "npm run build"   // if applicable
  },
  "engines": {
    "node": "18.x"
  }
}

Step 4: Set Environment Variables

Your API keys and secrets must never be committed to your repo. Set them on the platform instead.

# Vercel
vercel env add OPENAI_API_KEY

# Railway
railway variables set OPENAI_API_KEY=sk-...

# Fly.io
fly secrets set OPENAI_API_KEY=sk-...

# Render
# Dashboard > Your Service > Environment > Add Environment Variable

# Replit
# Secrets tab (lock icon) > Add new secret

Set every environment variable your system needs. Common ones:

  • OPENAI_API_KEY (or your AI provider's key)
  • PORT (usually auto-set by the platform)
  • Any database URLs, webhook secrets, or third-party API keys

Step 5: Deploy

Push your code to the platform. Watch the build logs for errors.

# Vercel
vercel --prod

# Railway
railway up

# Fly.io
fly launch    # first time
fly deploy    # subsequent deploys

# Render
# Push to GitHub — auto-deploys if connected

# Replit
# Click "Run" then "Deploy"

If the build fails, read the error message carefully. The most common issues are missing dependencies and incorrect start commands. Fix, commit, and redeploy.

Step 6: Verify

Open your live URL and run through the full flow.

  1. Open the live URL in your browser (or send a request with curl)
  2. Provide real input — the same input you used in your eval tests
  3. Confirm the output matches what you see locally
  4. Open the URL in an incognito/private window to confirm it works without your local cookies or session
  5. If possible, run your eval suite against the deployed URL
# Quick verification with curl (for API deployments)
curl -X POST https://your-app.railway.app/api/run \
  -H "Content-Type: application/json" \
  -d '{"input": "your test input"}'

Step 7: Record Your Deployment

Update your README with the live URL and commit.

# Add to your README
echo "\n## Live URL\nhttps://your-app.railway.app\n" >> README.md
git add .
git commit -m "Deploy to [platform] — live at [URL]"
git push

Common Deploy Issues

Issue Platform Fix
Build fails All Check build logs. Confirm all dependencies are in package.json or requirements.txt. Run the build command locally first.
Missing env vars All Go to the platform dashboard and set each variable. The app will crash on startup if a required key is missing.
Port mismatch Railway, Fly.io Use the PORT environment variable in your code. Do not hardcode a port number. Example: app.listen(process.env.PORT || 3000)
Timeout on AI calls Vercel Vercel serverless functions have a 10-second timeout on free tier. Use maxDuration in your route config, or switch to Railway for long-running requests.
Cold start too slow Vercel, Render Expected on free tiers. The first request after inactivity will be slow. Note this in your demo — it is normal.
Module not found All A dependency is missing from your package file. Run pip freeze > requirements.txt or check package.json dependencies.
CORS error in browser All Add CORS headers to your API. FastAPI: from fastapi.middleware.cors import CORSMiddleware. Express: app.use(cors()).

Done Criteria

  • Platform account created and CLI installed (if applicable)
  • Project configured with correct build and start commands
  • All environment variables set on the platform
  • Build succeeds without errors
  • System is accessible at a live public URL
  • Core flow works on the deployed version with real input
  • Verified from incognito window or second device
  • Live URL added to README
  • Deployment record filled out
  • Code committed and pushed
