Module 01: Kickoff

What You Are Building

In this module, you are orienting yourself to the Ship Systems course. By the end, you will have a clear picture of what you are building across all 9 modules, what tools you need, and a personal commitment to the rules that make this work. Your output is a completed Ready to Ship checklist confirming you have everything in place to start building.

Why It Matters

Most people who want to build AI systems get stuck in research mode. They read documentation, watch tutorials, and tinker with prompts—but never ship anything real. This course exists to break that pattern. Over 9 modules, you will go from a blank page to a deployed, working AI system with a recorded demo. Every module produces a concrete artifact. Nothing is theoretical. If you set up correctly now, the rest of the course flows. If you skip this step, you will hit friction later. Take 20 minutes to get your environment and mindset right.

Step-by-Step Instructions

Step 1: Understand What You Will Build

Across the 9 modules of Ship Systems, you are building a complete AI system from scratch. Not a toy. Not a demo. A working system that takes real inputs, processes them through AI, and produces real outputs for a real user.

Here is the full roadmap:

Step 2: Gather What You Need

Before you proceed to Module 02, make sure you have the following ready:

  • A computer with a code editor installed (VS Code recommended)
  • A terminal you are comfortable using
  • Git installed and a GitHub account
  • Node.js or Python installed (depending on your preference)
  • An AI API key — OpenAI, Anthropic, or another provider you plan to use
  • Claude or ChatGPT access for brainstorming and drafting (used heavily in Modules 02-03)
  • A quiet block of time — plan for 8-12 hours total across all 9 modules

Step 3: Commit to the Rules

These three rules keep the course working. Read them. Internalize them.

Rule 1: Ship over polish. A working system that is rough beats a polished system that does not exist. You can always improve later. Your job in this course is to get something real into the world.

Rule 2: Every module produces an artifact. You do not move to the next module until you have the output for the current one. No skipping. No "I'll come back to it." Each artifact builds on the last.

Rule 3: Ask for help when stuck. If you are stuck for more than 15 minutes, use Claude or ChatGPT to get unstuck. Paste your current state, describe the problem, and ask for a concrete next step. Do not spin.

Step 4: Set Up Your Project Folder

Create a project directory for your AI system. You will use this throughout the entire course.

mkdir my-ai-system
cd my-ai-system
git init
mkdir docs
echo "# My AI System" > README.md
git add .
git commit -m "Initial commit"

Replace my-ai-system with a name that describes what you plan to build. If you do not know yet, that is fine—use a placeholder. You will define it in Module 02.

Step 5: Complete the Ready to Ship Checklist

Go through the artifact template below. Check every item honestly. If something is missing, fix it now before moving on.

Artifact: Ready to Ship Checklist

Copy this into a file called docs/ready-to-ship.md in your project folder and fill it out:
