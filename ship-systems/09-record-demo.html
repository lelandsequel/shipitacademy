Module 09: Record Demo

What You Are Building

A 2-minute demo video of your AI system. This is your proof of work: a shareable recording that shows what you built, how it works, and what comes next. When this module is done, you will have a portfolio-ready video linked in your project README.

Why It Matters

A 2-minute demo is worth more than a 20-page document. It is proof you can build. It is a portfolio piece you can send to anyone — a hiring manager, a collaborator, an investor, a friend. Code in a repo is invisible to most people. A demo video makes your work real and legible. This is how you show what you shipped.

Step-by-Step Instructions

Step 1: Understand the Demo Structure

Your demo must follow this strict 2-minute format. Every second counts.

Segment Duration What to Cover
What it does ~20 sec One sentence: what the system does and who it is for.
Live demo ~60 sec Screen recording: real input, real output, on the live deployed system.
How it works ~20 sec Quick architecture: which model, which tools, where it is deployed.
What is next ~20 sec One or two features you would build next with more time.

Step 2: Write Your Script

You are not writing a word-for-word script. You are writing bullet points for each segment so you know exactly what to say and show. Copy the template below and fill it in.

Demo Script Template

WHAT IT DOES (~20 sec)
- "[System name] is a [type of system] that [what it does] for [who]."
- "You give it [input] and it [produces output]."

LIVE DEMO (~60 sec)
1. Open [your live URL]
2. Enter [specific input you will use]
3. Click [button / run command]
4. Show the system processing
5. Show the output
6. Point out [one thing that works particularly well]

HOW IT WORKS (~20 sec)
- "Under the hood, it uses [model/API] for [task]."
- "[Tool/framework] handles [task]."
- "It is deployed on [platform]."

WHAT IS NEXT (~20 sec)
- "With more time, I would add [feature 1]."
- "I would also [feature 2]."
- "The system is designed to [future capability]."

Step 3: Set Up Screen Recording

Choose a screen recording tool. You need screen capture with voiceover (microphone audio). Face cam is optional.

Tool Platform Cost Notes
Loom Mac, Windows, Chrome Free (up to 5 min) Easiest option. Records, hosts, and generates a share link automatically.
QuickTime Player Mac Free (built-in) File > New Screen Recording. Select microphone in the dropdown.
OBS Studio Mac, Windows, Linux Free Most powerful. Add a Display Capture source and an Audio Input Capture source.
Xbox Game Bar Windows Free (built-in) Press Win+G. Works for any window, not just games.

Recording settings:

  • Resolution: 1080p minimum (1920x1080)
  • Audio: use a headset mic or external microphone if possible. Avoid laptop mic in noisy environments.
  • Close all unnecessary tabs and notifications before recording
  • Increase your font size / zoom level so text is readable in the recording

Step 4: Do a Dry Run

Run through the entire demo once without recording. Time yourself.

  • If you are over 2 minutes and 30 seconds, cut content. Remove the least essential point from each segment.
  • If you are under 1 minute and 30 seconds, add detail to the live demo segment. Show a second example input.
  • Make sure your live system is working. Open the URL and verify it responds before you start recording.
  • Pre-type your demo input so you can paste it during the recording instead of typing live.

Step 5: Record

Record the demo. Give yourself 2 to 3 takes maximum. Pick the best one.

  1. Start the screen recording tool
  2. Open your deployed system URL in the browser
  3. Begin speaking: "This is [system name]..."
  4. Walk through each segment of your script
  5. Stop the recording

No editing is required. A clean single take is better than a heavily edited video. If you stumble on a word, keep going. Authenticity matters more than polish.

Step 6: Upload and Share

Upload your video to a platform that generates a shareable link.

  • Loom: automatically generates a share link after recording
  • YouTube: upload as Unlisted. Copy the link.
  • Google Drive: upload the file, set sharing to "Anyone with the link," copy the link.

Add the demo link to your project README:

## Demo Video
[Watch the 2-minute demo](https://your-demo-link-here)

git add README.md
git commit -m "Add demo video link to README"
git push

Artifact Template

Copy the script template from Step 2 above. Here is the completed version as an example.

Example: Completed Demo Script

WHAT IT DOES (~20 sec)
- "ContractScan is an AI-powered contract analysis tool that identifies
  legal risks for freelancers and small business owners."
- "You paste in a contract and it highlights the clauses you should
  negotiate before signing."

LIVE DEMO (~60 sec)
1. Open https://contractscan.up.railway.app
2. Paste a sample freelance contract into the input field
3. Click "Analyze"
4. Show the loading spinner while the system processes
5. Show the output: three risks identified with section references
6. Point out that it catches the auto-renewal clause that most
   people miss

HOW IT WORKS (~20 sec)
- "Under the hood, it uses GPT-4 for clause analysis."
- "FastAPI handles the API layer."
- "It is deployed on Railway with a Streamlit frontend."

WHAT IS NEXT (~20 sec)
- "With more time, I would add PDF upload support so users do not
  have to copy-paste."
- "I would also add a comparison mode to diff two contract versions."
- "The system is designed to eventually handle multi-party agreements."

Done Criteria

  • Demo script written with bullet points for all four segments
  • Dry run completed and timed at 2 minutes or less
  • Video recorded with screen capture and clear voiceover audio
  • Demo shows the live deployed system, not localhost
  • Demo uses real input and produces real output
  • Video is 2 minutes or less
  • Video uploaded to a shareable platform (Loom, YouTube, Google Drive)
  • Demo link added to project README
  • Final commit pushed to repo
